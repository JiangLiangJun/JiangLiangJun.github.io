使用Github搭建Hexo个人博客

发表于 2017-11-19 | 分类于 博客 | 阅读次数:
字数统计: 726 | 阅读时长 ≈ 3

由于博客搭建完成好几天了,一想到有挺多的喜欢搭建个人博客的小伙伴被重重难题迷倒,而放弃了搭建博客的兴趣,所以我在这里写出一篇关于搭建个人博客的教程(附带其他博主的帖子).

由于搭建博客需要一定的耐心和一定的阅读理解能力,所以在搭建博客的过程中难免会遇到难题.请不要急躁,慢慢来.跟着我的步骤肯定能够成功的.

在这里我先引用我之前搭建的博客的一篇帖子使用 Github 空间搭建 Hexo 技术博客–安装篇（基于 IntelliJ IDEA）,在里面有一些没有提到的细节,我会在这里都写出来.

由于我们是用Github搭建Hexo个人博客所以必须得有一个Github账号,而DNSPOD这是一个购买域名的网站(当然,你也可以用腾讯云和阿里云这些类似的网站!!!),而搭建所需软件:我只安装了git和node.js,其实安全三者集合包也没啥区别…

部署开始部分可以直接跟着帖子进行,当你安装软件后,验证安装出现不同的版本号是不会影响接下来搭建的步骤.

项目的目录可以直接修改存放的位置,文件夹名字可以自己选取(不能使用中文),而hexo目录的命名必须一样.

主题方面我们可以先使用教程的主题,后面搭建主题完成之后可以自己随意的修改.以后修改不同的主题时,必须进入hexo目录下的_config.yml文件里面修改theme的主题名字(冒号后面带有空格)

注意创建账号的时候账号登录名必须和仓库名字一致(注意不可以使用中文)

在使用Github部署的时候,使用hexo deploy命令时候,可能不会出现提示框,而会在git命令行里面出现(yes/no),这个时候就输入yes就行.

由于每个人购买的域名不一样,所有需要个人手动去自己购买域名的网站进行域名解析.解析的过程中有两个A的记录类型,而主机记录修改为 * 和 @ ,而不是code.(下面放出例图)

而最后提到使用 IntelliJ IDEA则是一款编程软件,而我使用的是JetBrains WebStorm 和 PhpStorm.

在这里我非常感谢帖子的博主,让我搭建好了博客!!!
